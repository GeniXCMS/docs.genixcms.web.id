    File & Image Uploading¶

    Handling file and image uploads in GeniXCMS can be done in two primary ways depending on your use case:

    1. Programmatically via PHP using the native Upload::go() method (ideal for custom forms, profile pictures, or plugin-specific logic).
    2. Visually via the UI using the built-in elFinder media manager (ideal for rich text editors or choosing existing files).

    1. Uploading Programmatically (Upload::go())¶

    The Upload class provides a streamlined, secure way to handle standard HTML <input type="file"> submissions in your controllers or modules.

    HTML Form Setup¶

    Always ensure your form has enctype="multipart/form-data".

    <form action="" method="post" enctype="multipart/form-data">
    <div class="mb-3">
        <label>Upload Profile Picture</label>
        <input type="file" name="avatar_image" class="form-control" accept="image/png, image/jpeg">
    </div>
    <button type="submit" name="submit_upload" class="btn btn-primary">Upload</button>
</form>

    Processing the Upload¶

    In your PHP logic, call Upload::go() to process the file. 

    if (isset($_POST['submit_upload'])) {
    // Define allowed extensions
    $allowed = ['jpg', 'jpeg', 'png', 'gif', 'webp'];

    // Define the upload path relative to the root (e.g., 'assets/images/avatars/')
    $path = '/assets/images/avatars/';

    // Execute the upload
    // Parameters: input_name, path, allowed_exts, use_unique_name
    $upload = Upload::go('avatar_image', $path, $allowed, true);

    if (isset($upload['error']) && $upload['error'] != '') {
        // Handle Error
        echo "<div class='alert alert-danger'>{$upload['error']}</div>";
    } else {
        // Success
        $filename = $upload['filename'];
        $fullPath = $upload['filepath'];
        $url      = $upload['fileurl'];

        echo "<div class='alert alert-success'>File uploaded to: {$url}</div>";

        // You can now save $url to your database
    }
}

    Upload::go() Syntax¶

    Upload::go(
    string $input,     // The 'name' attribute of your <input type="file">
    string $path,      // Path to save the file (e.g., '/assets/images/')
    array $allowed,    // Array of allowed extensions (e.g., ['png', 'jpg'])
    bool $uniq = true, // If true, adds a unique hash to prevent overwriting
    int $size = null,  // (Optional) Max file size limit
    int $width = null, // (Optional) Max width for images
    int $height = null // (Optional) Max height for images
);

    2. Uploading via the UI (elFinder)¶

    GeniXCMS bundles the elFinder file manager as its core media library. It provides a full GUI for users to browse, upload, and manage files visually.

    It is natively integrated into editor instances (like GxEditor and TinyMCE), but you can also spawn it independently.

    Spawning an elFinder Popup¶

    If you are writing a custom module or theme options panel and want users to "Select an Image" from the media library (or upload a new one), you can trigger the elFinder dialog via AJAX.

    1. Create the trigger element and input field: 

    <div class="input-group">
    <input type="text" id="my_image_input" name="image_url" class="form-control" placeholder="Select an image...">
    <button type="button" class="btn btn-secondary" onclick="openMediaManager()">
        <i class="bi bi-folder2-open"></i> Browse
    </button>
</div>

    2. Trigger elFinder using .dialogelfinder Widget: Note: The url parameter must point to the secure elFinder AJAX endpoint, which requires an active session Token for security.

    <script>
function openMediaManager() {
    // We use the native jQuery UI Dialog wrapper for elFinder
    var fm = $("<div/>").dialogelfinder({
        // Generate the secure URL using GeniXCMS Url & Token helpers
        url: "<?= Url::ajax('elfinder/' . Token::create()); ?>",
        lang: "en", 
        width: 840, 
        height: 450,
        destroyOnClose: true,
        // The callback triggered when a user double-clicks an image
        getFileCallback: function (files, fm) {
            // Put the URL into the input field
            $("#my_image_input").val(files.url);

            // Optional: Update an image preview tag
            // $("#post_image_preview").attr("src", files.url).removeClass("d-none");
        },
        commandsOptions: { 
            getfile: { 
                oncomplete: "close", // Close dialog after selection
                folders: false       // Prevent selecting folders
            } 
        }
    }).dialogelfinder("instance");
}
</script>

    ⚙️ Zero-Backend Upload Processing (How It Works)¶

    One of the biggest advantages of using the elFinder integration is that you do not need to write any backend PHP code to handle the file upload. When you trigger the File Browser using the widget above, the user can visually browse, select, or upload new files directly.

    Ketika user melakukan proses upload melalui popup tersebut, proses upload secara terprogram akan di-handle otomatis oleh GeniXCMS di belakang layar:

    1. The client sends an AJAX upload request securely.
    2. The request routes to inc/mod/elfinder/elfinder-ajax.control.php (The elFinder Backend Controller).
    3. The controller validates your active Session Token to block CSRF and unauthorized access.
    4. The backend securely executes the physical file upload directly into the assets/ directory.
    5. The final public URL is returned to the client and injected automatically into your target input field (e.g., #my_image_input) via the getFileCallback.

    This means you only need to save the final image_url string to your database, significantly speeding up development time!

    3. Background API Uploads (Headless elFinder)¶

    There are scenarios where you want users to click a custom "Add Image" button layout or drop a file into an editor without ever seeing the file manager dialog popup.

    You can accomplish this programmatically by sending an AJAX request directly to the elFinder Controller (using its API) with your file payload.

    Custom HTML Trigger¶

    <button type="button" onclick="document.getElementById('hidden_file_input').click()" class="aspect-video rounded-2xl border-2 border-dashed border-slate-200 flex flex-col items-center justify-center text-slate-400 hover:border-primary hover:text-primary hover:bg-primary/5 transition-all outline-none">
    <span class="material-symbols-outlined text-[32px]">add_a_photo</span>
    <span class="text-[10px] font-black uppercase tracking-widest mt-2">Add Image</span>
</button>
<input type="file" id="hidden_file_input" class="d-none" onchange="uploadDirectAsset(this.files[0])">

    The AJAX Upload Logic¶

    The function first makes an init request to retrieve the current working directory (cwd.hash), then posts the file content payload.

    <script>
function uploadDirectAsset(file) {
    if (!file) return;

    // Secure URL mapping to the elFinder endpoint
    var elfinderUrl = "<?= Url::ajax('elfinder/' . Token::create()); ?>";
    var sep = elfinderUrl.indexOf('?') === -1 ? '?' : '&';

    // Step 1: Initialize the API to get the target directory hash
    $.ajax({ url: elfinderUrl + sep + 'cmd=open&init=1&target=', type: 'GET', dataType: 'json' })
     .done(function(initData) {
         if (!initData || !initData.cwd) {
             console.error('Failed to init elfinder API');
             return;
         }

         // Extract the hash ID of the root directory
         var targetHash = initData.cwd.hash;

         // Step 2: Build the multipart payload
         var fd = new FormData();
         fd.append('cmd', 'upload');
         fd.append('target', targetHash);
         fd.append('upload[]', file);

         // Step 3: Execute the background upload
         $.ajax({
             url: elfinderUrl, 
             data: fd, 
             cache: false, 
             contentType: false, 
             processData: false, 
             type: 'POST',
             success: function(data) {
                 var parsed = typeof data === 'string' ? JSON.parse(data) : data;

                 // If successful, 'added' array contains the new file metadata
                 if (parsed.added && parsed.added.length > 0) {
                     var fileUrl = parsed.added[0].url;
                     console.log("Upload Success: ", fileUrl);

                     // Example: Insert the image into the DOM
                     // $("#preview_image").attr('src', fileUrl);
                 } else if (parsed.error) {
                     console.error("Upload Error: ", parsed.error);
                 }
             },
             error: function(jqXHR, textStatus, errorThrown) {
                 console.error(textStatus + " " + errorThrown);
             }
         });
     });
}
</script>
