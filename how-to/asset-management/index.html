    Asset Management with Asset Class¶

    The Asset class in GeniXCMS is a modern replacement for manually echoing script and link tags in hooks. It provides a centralized registry for CSS and JS files, simplifies dependency management, and supports context-aware loading (frontend vs. backend).

    1. Registering Assets¶

    Use Asset::register() to inform the system about a file. You can register assets from a module's init() or a theme's constructor.

    Syntax¶

    Asset::register(
    string $id, 
    string $type, 
    string $url, 
    string $pos = 'header', 
    array $deps = [], 
    int $priority = 20, 
    string $context = 'all'
);
    • $id: A unique identifier (e.g., bootstrap, my-custom-js).
    • $type: css, js, or raw.
    • $url: The full URL to the file.
    • $pos: header or footer.
    • $deps: Array of dependency IDs (e.g., ['jquery']).
    • $priority: Execution order (lower runs first).
    • $context: admin, frontend, or all. This determines where the asset is allowed to load.

    Example: Registering from a Module¶

    public static function init() {
    $siteUrl = Site::$url;
    // CSS for admin only
    Asset::register('my-mod-css', 'css', $siteUrl . '/inc/mod/my-mod/assets/style.css', 'header', [], 20, 'admin');

    // JS for both frontend and backend
    Asset::register('my-mod-js', 'js', $siteUrl . '/inc/mod/my-mod/assets/script.js', 'footer', ['jquery'], 25, 'all');
}

    2. Enqueueing Assets¶

    Once registered, an asset must be "enqueued" to be rendered in the current request.

    // Enqueue by ID
Asset::enqueue('my-mod-js');

// Enqueue multiple
Asset::enqueue(['my-mod-css', 'bootstrap']);

    If an asset is enqueued with a context that doesn't match the current environment (e.g., an admin asset enqueued on the frontend), the Asset manager will automatically skip it.

    3. Dependency Management¶

    The Asset manager automatically resolves dependencies. If you enqueue an asset that depends on jquery, the system will ensure jquery is rendered before your script, even if jquery wasn't explicitly enqueued.

    Pre-defined Core Assets¶

    ID Type Version
    jquery JS 3.7.1
    bootstrap JS/CSS 5.3.2
    font-awesome CSS 6.5.1
    bi CSS Bootstrap Icons 1.11.3

    4. Context-Aware Loading¶

    GeniXCMS 2.0 introduces explicit contexts to prevent admin-specific scripts from bloat-loading on the frontend.

    • admin: Only loads if GX_ADMIN is defined as true.
    • frontend: Only loads if GX_ADMIN is not defined or false.
    • all: Always allowed to load.

    5. Integrating with Themes¶

    Themes must call the following methods in their header and footer templates to output enqueued assets. If your theme uses Latte (which is recommended), you must use the |noescape filter to prevent HTML entities from being escaped.

    header.php / header.latte¶

    Native PHP: 

    <head>
    ...
    <?= Site::loadLibHeader(); ?>
</head>
    Latte Template: 
    <head>
    ...
    {Site::loadLibHeader()|noescape}
</head>

    footer.php / footer.latte¶

    Native PHP: 

        ...
    <?= Site::loadLibFooter(); ?>
</body>
    Latte Template: 
        ...
    {Site::loadLibFooter()|noescape}
</body>

    Note: These methods now return the generated HTML strings instead of echoing them directly, ensuring that metadata and layouts maintain the correct sequence.

    6. Raw Assets¶

    You can also register "raw" HTML blocks (like inline <script> or <style> tags).

    $rawHtml = '<script>console.log("Hello GeniXCMS!");</script>';
Asset::register('my-inline-js', 'raw', $rawHtml, 'footer', [], 100);
Asset::enqueue('my-inline-js');

    7. Offline Mode (v2.4.0+)¶

    GeniXCMS supports a fully offline-capable admin panel. When OFFLINE_MODE=true in config.php, all framework assets are served from local files instead of CDN.

    Enabling Offline Mode¶

    // inc/config/config.php
!defined('OFFLINE_MODE') ? define('OFFLINE_MODE', true) : null;

    What Gets Localized¶

    Asset Local Path
    jQuery 3.7.1 assets/js/vendor/jquery-3.7.1.min.js
    jQuery UI 1.13.2 assets/js/vendor/jquery-ui-1.13.2.min.js
    Bootstrap 5.3.3 CSS assets/css/vendor/bootstrap-5.3.3.min.css
    Bootstrap 5.3.3 JS assets/js/vendor/bootstrap-5.3.3.bundle.min.js
    Bootstrap Icons 1.11.3 assets/css/vendor/bootstrap-icons-1.11.3.min.css
    Font Awesome 6.7.2 assets/fonts/fontawesome/css/all.min.css
    Inter Font (300–900) assets/fonts/inter/inter-*.woff2
    Chart.js 4.4.1 assets/js/vendor/chartjs-4.4.1.min.js
    jsVectorMap 1.5.3 assets/js/vendor/jsvectormap-1.5.3.min.js
    EditorJS + tools assets/js/vendor/editorjs/

    Registering Your Own Offline-Aware Assets¶

    $offline = defined('OFFLINE_MODE') && OFFLINE_MODE;
$siteUrl = rtrim(Site::$url, '/');

Asset::register(
    'my-lib',
    'js',
    $offline
        ? $siteUrl . '/assets/js/vendor/my-lib.min.js'
        : 'https://cdn.example.com/my-lib.min.js',
    'footer',
    [],
    20,
    'all'
);
