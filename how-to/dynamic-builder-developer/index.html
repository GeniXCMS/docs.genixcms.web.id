    Dynamic Builder Developer Guide¶

    This document describes how to extend and integrate the Dynamic Builder module for GeniXCMS from a technical perspective.

    Integration Architecture¶

    The Dynamic Builder is implemented as a GeniXCMS module using the GrapesJS framework. It is registered in the admin dashboard through AdminMenu::add() and is accessible at: index.php?page=mods&mod=dynamic-builder.

    Key Technical Components¶

    • Core Module: Located at inc/mod/dynamic-builder/.
    • Primary Logic: JavaScript found at assets/js/builder.js.
    • Layout Management: layout.php handles the initialization and CSS/JS injection.
    • Canvas Styles: Custom CSS for the editor iframe is defined in assets/css/canvas.css.

    Integration via Hooks¶

    The module is integrated into the core Page management system via standard GeniXCMS Hooks:

    • admin_footer_action: Used to load the editor's modal layout and dependency scripts only on the specific Add/Edit Page routes.
    • page_param_form_bottom: Used to inject the primary UI trigger ("Launch Visual Editor") at the bottom of the Page form.
    • header_load_lib: Enqueues necessary CSS and Google Fonts by scanning the page content for specific font families.
    • footer_load_lib: Loads standard frontend JavaScript required for interactive components (like carousels and counters).

    Extending the Builder¶

    The builder.js file contains all the configurations for blocks, component types, and commands.

    1. Registering Custom Blocks¶

    To add a new Block to the sidebar, use the getPremiumBlocks function or append directly to the GrapesJS BlockManager:

    bm.add('my-custom-block', {
    label: 'Custom Block Title',
    category: 'Your Category',
    content: {
        type: 'my-custom-type',
        components: [{ /* initial DOM structure */ }]
    }
});

    2. Defining Component Types¶

    Use editor.DomComponents.addType to define custom behaviors and traits for your elements:

    editor.DomComponents.addType('my-custom-type', {
    model: {
        defaults: {
            name: 'Custom Component',
            draggable: true,
            droppable: true,
            traits: [
                { type: 'text', label: 'Custom Attribute', name: 'data-custom' }
            ],
            // Scripts to run inside the canvas
            script: function() {
                console.log('Component initialized in canvas');
            }
        }
    }
});

    3. Implementing Toolbar Commands¶

    Extend a component's functionality with custom toolbar buttons and commands:

    editor.Commands.add('my-custom-command', {
    run(editor, sender) {
        const selected = editor.getSelected();
        if (selected) {
            // Perform DOM manipulation or logic here
            selected.append('<div>New element</div>');
        }
    }
});

    Best Practices¶

    • Bootstrap 5 Classes: Ensure all blocks use standard Bootstrap 5 classes to maintain compatibility with the main GeniXCMS core styling.
    • Component Specificity: Use type: 'text' for editable text elements and type: 'image' for images to ensure they are correctly handled by GrapesJS's built-in managers.
    • Isolate Canvas Styles: If your components require custom styling, add it to canvas.css to ensure it is rendered within the editor's iframe.
    • Serialization: The builder synchronizes content back to a standard textarea in the GeniXCMS Pages or Posts admin view, including both the generated HTML and a <style> block containing the component CSS.

    CSS Implementation for Ratios¶

    Custom aspect ratios (like 3:4 for team photos) should be defined in canvas.css to be correctly rendered within the editor:

    .ratio-3x4 {
    --bs-aspect-ratio: 133.33%;
}

    Data Serialization¶

    The save-builder-page command handles the data synchronization:

    1. It extracts editor.getHtml() and editor.getCss().
    2. Consolidates them into a single string: <style>...</style><html>...</html>.
    3. Sets the value of the primary GeniXCMS editor textarea for persistence.
