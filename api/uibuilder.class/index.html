    UiBuilder Class API Reference¶

    The UiBuilder class is a high-level UI orchestration engine in GeniXCMS. It allows developers to define complex administrative interfaces using structured PHP arrays (schemas), which are then rendered into professional, responsive HTML.

    🏗️ Class Initialization¶

    new UiBuilder(array $schema = [])¶

    Initializes a new instance of the builder with a given UI schema.

    • $schema: An associative array defining headers, tabs, and layout elements.
    • Behavior: The constructor automatically detects the active tab from the URL &tab= parameter or falls back to the default_tab defined in the schema.

    ⚡ Public Methods¶

    renderHeader(): void¶

    Renders the standardized GeniXCMS module header.

    • Output: Generates a sticky-top row containing the module title, subtitle, version badge, and action buttons.
    • Shrink Logic: Automatically injects the JavaScript needed to shrink the header when the user scrolls down, preserving vertical screen real estate.
    • Note: This method is called internally by render(), but can be called manually if you only need the header.

    render(): void¶

    The primary entry point to generate the full module interface.

    • Functionality:
      1. Wraps everything in a container-fluid.
      2. Calls renderHeader().
      3. Renders the tab navigation bar (if tabs are present).
      4. Orchestrates visibility: For tab_mode => 'js', it renders all panes; for link mode, it only renders the active tab's content.
      5. Renders any global content or modals defined outside the tab system.

    renderElement(array $el, bool $return = false): string¶

    Recursive engine used to render individual UI components.

    • $el: The individual element definition array (e.g., ['type' => 'card', ... ]).
    • $return:
      • false (default): Echoes the HTML directly to the output buffer.
      • true: Returns the rendered HTML as a string (uses ob_start() internally).
    • Supported Elements: Handles everything from simple inputs to complex nested grids, loops, and statistical visualizations.

    📋 Root Schema Reference¶

    When defining your $schema, the following top-level keys control the UiBuilder behavior:

    Key Type Description
    header array Page identity: title, subtitle, icon, version, and buttons.
    tabs array A list of tab definitions. Each tab must have a label and content.
    tab_mode string link (reloads page) or js (client-side switch).
    tab_style string pills (default), tabs, or modern (underlined).
    card_wrapper bool If true, the entire tab content area is wrapped in a shadowed card.
    default_tab string The ID of the tab to show by default.
    content array Global elements rendered at the bottom (e.g., Modals).

    🎨 Card Element Extended Options¶

    The card element ('type' => 'card') supports advanced layout properties:

    Property Type Description
    title string The title displayed in the card header.
    icon string Bootstrap icon class for the header (e.g., bi bi-gear).
    no_padding bool Removes padding from .card-body. Ideal for tables.
    full_height bool Adds h-100 class to the card.
    header_action string Raw HTML/Elements to be placed on the right side of the card header.
    footer string Raw HTML content for the card footer.
    footer_class string Custom CSS classes for the card footer (e.g., bg-transparent border-0).
    footer_no_padding bool Removes internal padding from the footer container.

    The following table lists all element types supported by the renderElement() engine:

    Element Type Description Key Properties

    textarea Multi-line text area (supports Rich Editors). name, label, value, rows, class

    ⚡ Reactivity & Dependencies (require)¶

    UiBuilder supports element dependencies, allowing you to show/hide or enable/disable elements based on the state of another element.

    Dependency Parameters¶

    Property Type Description
    require string The name or id of the target element to watch.
    require_value string The value that must be met for the dependency to trigger. Default: on.
    require_action string The action to perform: show (default) or enable.

    Usage Examples¶

    1. Show field based on Select value¶

    [
    'type' => 'select',
    'name' => 'payment_method',
    'label' => 'Payment Method',
    'options' => ['cash' => 'Cash', 'transfer' => 'Bank Transfer']
],
[
    'type' => 'input',
    'name' => 'bank_name',
    'label' => 'Bank Name',
    'require' => 'payment_method',
    'require_value' => 'transfer'
]

    2. Enable input based on Checkbox¶

    [
    'type' => 'checkbox',
    'name' => 'use_custom_discount',
    'label' => 'Use Custom Discount'
],
[
    'type' => 'input',
    'name' => 'discount_value',
    'label' => 'Discount Percentage',
    'require' => 'use_custom_discount',
    'require_value' => 'on',
    'require_action' => 'enable'
]

    💡 Practical Pattern: The "Double Render"¶

    If you need to wrap the UiBuilder inside a form or custom container, use the instance call within your backend controller:

    // 1. Define Schema
$schema = [ ... ];

// 2. Initialize Builder
$builder = new UiBuilder($schema);

// 3. Render inside custom HTML
echo '<form method="POST" action="...">';
$builder->render(); 
echo '</form>';
