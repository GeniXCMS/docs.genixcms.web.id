    Posts Class¶

    The Posts class manages all types of content entries in GeniXCMS, including blog posts and static pages. It handles insertion, retrieval, metadata management (params), and automated excerpt generation.

    Base Operations¶

    insert()¶

    Adds a new content entry. Automatically handles slug creation, excerpt caching, and hook triggers. 

    Posts::insert([
    'title'   => 'GeniXCMS is Awesome',
    'content' => 'Full article content...',
    'author'  => 'admin',
    'cat'     => 5,
    'type'    => 'post',
    'status'  => '1',
    'date'    => date('Y-m-d H:i:s')
]);

    update()¶

    Updates an existing record. Requires the post ID to be present in $_GET['id'] (admin context) or passed within the logic. Posts::update(array $vars);

    delete()¶

    Completely removes a post, its metadata (params), and associated comments. Posts::delete(int $id);

    Content Display Helpers¶

    content()¶

    Returns cleaned post content. It strips legacy [[--readmore--]] tags and applies post_content_filter hooks. echo Posts::content($raw_content);

    format()¶

    Processes content for summary lists. If the [[--readmore--]] tag is present, it returns the text before the tag followed by a "Read More" link. echo Posts::format($raw_content, $post_id);

    excerpt()¶

    Retrieves a plain-text summary of the post. It uses a cached version from the database or generates a new one (caching it automatically) if missing. echo Posts::excerpt($post_id, $optional_content);

    Metadata (Parameters)¶

    Metadata allows you to attach custom attributes (like featured images, SEO tags, or layout settings) to any post.

    getParam()¶

    Posts::getParam(string $key, int $post_id);

    getParams()¶

    Retrieves all parameters (metadata) associated with a post as an associative array. $meta = Posts::getParams(int $post_id);

    addParam() / editParam()¶

    Posts::addParam($key, $value, $post_id); Posts::editParam($key, $value, $post_id);

    existParam()¶

    Posts::existParam(string $key, int $post_id);

    Retrieval & UI¶

    recent()¶

    Fetches a list of latest posts. 

    $posts = Posts::recent([
    'num'  => 5,
    'cat'  => 2,      // Optional category filter
    'type' => 'post'  // Default: 'post'
]);

    lists()¶

    Renders a complex HTML list of posts (Bootstrap compatible). 

    echo Posts::lists([
    'num'        => 5,
    'excerpt'    => true,
    'image'      => true,
    'image_size' => 150
]);

    dropdown()¶

    Generates an HTML <select> menu of content titles. echo Posts::dropdown(['name' => 'featured_post', 'type' => 'post']);

    getPageList()¶

    Returns an associative array of all published pages (ID => Title). Ideal for configuration dropdowns. $pages = Posts::getPageList();

    🖥️ Dashboard UI Helpers¶

    These methods are used to generate the standardized GeniXCMS dashboard tables, supporting both AJAX and traditional rendering.

    getDashboardHeaders()¶

    Returns the array of headers for a post type table, with support for the admin_posts_table_headers (or admin_pages_*) hook. Posts::getDashboardHeaders(string $postType);

    getDashboardRow()¶

    Generates a formatted row for a post object, including thumbnail, action buttons, and status badges. Supports the admin_posts_table_row hook. Posts::getDashboardRow(object $pObj, string $group, string $username, string $postType);

    getTypeLabel()¶

    Retrieves a specific UI label for a post type (e.g., 'new_item', 'repository_title') with fallback to defaults. Posts::getTypeLabel(string $type, string $key);

    Post Type Customization¶

    You can customize the labels displayed in the admin interface for different post types.

    setTypeLabel()¶

    Posts::setTypeLabel('portfolio', [
    'label' => 'Portfolio Item',
    'create' => 'Add Project',
    'title_placeholder' => 'Enter project name...'
]);
